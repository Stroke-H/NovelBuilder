# NovelGenerater 部署文件

## 1. 项目结构

```text
NovelGenerater/
  apps/
    api/    # Go + Gin + MySQL
    web/    # Vue 3 + Vite + Pinia + Element Plus
  .agents/skills/
  changeLog/
  docs/
  scripts/
```

## 2. 运行端口

- 前端开发端口：`5174`
- 后端开发端口：`19081`

生产部署时建议：

- 前端静态资源：`80/443`
- 后端 API 内部端口：`19081`

## 3. 后端配置文件

本地配置目录：

- `apps/api/data/ai_config.json`
- `apps/api/data/database_config.json`

建议优先使用环境变量覆盖：

### 数据库

- `NOVEL_GENERATER_DB_CONFIG_FILE`
- `NOVEL_GENERATER_DB_DRIVER`
- `NOVEL_GENERATER_DB_HOST`
- `NOVEL_GENERATER_DB_PORT`
- `NOVEL_GENERATER_DB_USER`
- `NOVEL_GENERATER_DB_PASSWORD`
- `NOVEL_GENERATER_DB_NAME`

### AI

- `NOVEL_GENERATER_AI_CONFIG_FILE`

### 后端监听

- `NOVEL_GENERATER_BACKEND_HOST`
- `NOVEL_GENERATER_BACKEND_PORT`

## 6. 生产部署建议

### 前端

```bash
cd /Users/apple/NovelGenerater/apps/web
npm run build
```

产物目录：

- `apps/web/dist/`

建议由 Nginx 托管静态文件。

### 后端

```bash
cd /Users/apple/NovelGenerater/apps/api
go build -o novel-generater-api .
```

建议使用 systemd 或 supervisor 托管后端进程。

## 7. Nginx 反向代理示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /opt/NovelGenerater/apps/web/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:19081/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 8. 当前注意事项

1. `apps/api/data/*.json` 为本地真实配置，不应提交到 GitHub。
2. AI provider 配置已按原项目完整迁入，部署前要确认对应 key 可用。
3. 当前登录、小说生成、大纲续写、正文生成、审计和修订链路均依赖 MySQL 可连接。
